# Guardium TechZone Bootcamp Automation

Python automation framework for deploying and configuring Guardium bootcamp environments on IBM TechZone VPC Multi-VM.

## 🚀 Quick Start

```bash
# Install dependencies
pip install -r requirements.txt

# Run automation
python automation.py
```

## 📁 Structure

```
guardium_tz_bootcamp_automation/
├── automation.py              # Main orchestrator
├── config/config.yaml         # Configuration
├── core/                      # Framework modules
│   ├── config_loader.py      # Config & machines info
│   ├── ssh_client.py         # SSH operations
│   ├── state_manager.py      # Task state tracking

│   └── utils.py              # Utilities
```

## 🎯 Features

- **Multi-VM Deployment** - Automated setup across raptor, sauropod, ceratops
- **Database Installation** - Oracle 21c, MySQL, MongoDB with sample data
- **Network Configuration** - /etc/hosts, listeners, TNS
- **State Management** - Resume from failures
- **Stage Control** - Initial vs manual execution
- **SSH Operations** - Remote command execution, file transfers
- **Custom Variables** - Password and config from TechZone deployment

## 📋 Deployment Tasks

### Task 001: Setup /etc/hosts
Configures `/etc/hosts` on raptor with all VMs using private IPs and `demo.guardium` domain.

### Task 002: Deploy Oracle 21c on Sauropod
- Installs Oracle Database 21c Enterprise Edition
- Creates ORCLCDB container database with ORCLPDB1 pluggable database
- Configures listener, TNS, SQL*Net
- Installs HR sample schema in dedicated tablespace
- Installs and configures SQLcl
- Enables auto-start in `/etc/oratab`

**Credentials:** Uses `pwd` custom variable from TechZone deployment

### Task 003: Deploy MySQL on Raptor
- Installs MySQL 8.0
- Creates root user with authentication
- Configures for remote access
- Imports sample data

### Task 004: Deploy MongoDB on Raptor
- Installs MongoDB Enterprise 8.3
- Creates admin user with root role
- Enables authorization
- Imports sample data archive
- Creates `.mongo_env` with connection URI

## 🔧 Configuration

### machines_info.json
Auto-generated by TechZone post-deploy, contains:
```json
{
  "machines": [
    {"name": "raptor-xyz", "public_ip": "...", "private_ip": "..."},
    {"name": "sauropod-xyz", "public_ip": "...", "private_ip": "..."},
    {"name": "ceratops-xyz", "public_ip": "...", "private_ip": "..."}
  ],
  "custom_variables": {
    "pwd": "YourPassword123!",
    "stage": "initial_config"
  }
}
```

### Custom Variables
Define in TechZone manifest:
```yaml
variables:
  - name: pwd
    type: string
    description: "Password for deployed services"
    required: true
    sensitive: true
```

## 📖 Usage

```bash
# Run all tasks
python automation.py

# Continue after stage checkpoint
python automation.py --continue

# Check status
python automation.py --status

# Reset state
python automation.py --reset

# Stop at specific task
python automation.py --stop-at task_003
```

## 🔐 Security

- Passwords from `pwd` custom variable (sensitive)
- SSH keys for authentication
- Secure file permissions (600 for credentials)
- No hardcoded passwords

## 📝 Adding Tasks

```python
# tasks/my_task.py
def deploy_my_service(config: ConfigLoader, logger, verbose: bool = True) -> bool:
    """Deploy my service."""
    password = config.get_custom_variable('pwd')
    machine_ip = config.get_machine_ip('raptor', use_private=True)
    
    with SSHClient(host=machine_ip, password=password) as ssh:
        result = ssh.execute_command("echo 'Hello'")
        return result['rc'] == 0

# automation.py
from tasks.my_task import deploy_my_service
orchestrator.register_task(
    task_id="005_my_service",
    task_fn=lambda: deploy_my_service(config, logger),
    description="Deploy my service"
)
```

## 🐛 Troubleshooting

**SSH Connection Failed:**
```bash
# Check connectivity
ping <machine_ip>
ssh root@<machine_ip> -p 2223
```

**Task Failed:**
```bash
# Check logs
tail -100 logs/automation_*.log

# Reset and retry
python automation.py --reset
```

**State Corruption:**
```bash
rm state.json
python automation.py
```
